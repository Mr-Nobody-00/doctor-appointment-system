1 Architecture Overview

How all the pieces connect together

CLIENT (Browser - React on port 5173) | | User interacts with Login, Register, Dashboard, Admin Panel | React uses Axios to send HTTP requests | JWT Token attached in Authorization header | ▼ SERVER (Spring Boot on port 8080) | ├── CORS Filter (WebConfig.java) | Allows requests from localhost:5173 | ├── JWT Security Filter (JwtFilter.java) | Checks token → valid? → proceed : → 403 Forbidden | Public routes (/api/auth/**) skip this check | ├── Controller Layer (REST APIs) | AuthController, SpecialtyController, DoctorController | SlotController, AppointmentController | Receives requests, processes logic, returns JSON | ├── Repository Layer (Spring Data JPA) | UserRepo, SpecialtyRepo, DoctorRepo, SlotRepo, AppointmentRepo | Generates SQL automatically from method names | ├── Hibernate ORM | Converts Java objects ↔ SQL queries automatically | └── H2 Database (In-Memory) Tables: USERS, SPECIALTY, DOCTOR, SLOT, APPOINTMENT Data lives in RAM, resets on server restart

Architecture Pattern: This is a Layered Architecture — each layer has a single responsibility. Frontend (UI) → Controller (API logic) → Repository (data access) → Database (storage). Each layer only talks to the layer directly below it.

2 Frontend — React + Vite

The user interface layer that runs in the browser

What is React?

React is a JavaScript library for building user interfaces. It lets you build reusable components (like Login form, Doctor card, Appointment table) and compose them into full pages.

React is a Single Page Application (SPA) — the page never reloads. React Router swaps components in and out to simulate page navigation.

What is Vite?

Vite is the build tool. It bundles all your React files, serves them on localhost:5173, and provides instant hot-reload — when you save a file, the browser updates immediately without refreshing.

Key React Concepts Used

useState(initialValue) Creates a state variable. When state changes, the component re-renders. Example: useState('') for form inputs, useState([]) for lists.
useEffect(() => {}, []) Runs code when the component loads. The empty [] means "run once on page load". We use this to fetch data from the API when a page opens.
axios.get(url, { headers }) Sends a GET request to the backend to fetch data. The JWT token is sent in the headers for authentication.
axios.post(url, data, { headers }) Sends a POST request with JSON data to create something new (register user, book appointment, etc).
localStorage.setItem(key, value) Stores data in the browser's local storage. We store the JWT token here after login so it persists across page refreshes.
useNavigate() React Router hook for programmatic navigation. After login, we call navigate('/dashboard') to go to the dashboard page.

3 JWT Security

How we protect the API — only logged-in users can access it

What is JWT?

JSON Web Token is a compact, URL-safe token that contains encoded user information. It's like a digital ID card — the server creates it on login, and the client shows it with every request to prove their identity.

A JWT looks like: eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJ...

It has 3 parts separated by dots: Header.Payload.Signature

How JWT Works in Our App

1

User Sends Login Request

React sends POST /api/auth/login with email and password in the request body.

2

Server Validates Credentials

AuthController checks email and password against the database. If valid, calls JwtUtil.generateToken().

3

Server Creates JWT Token

JwtUtil encodes the user's email and role into a token, signs it with a secret key (HS256 algorithm), sets 24hr expiry, and returns it.

4

React Stores the Token

The token is saved in localStorage. For every future API call, React reads the token and puts it in the Authorization header: "Bearer eyJhbGci..."

5

JwtFilter Checks Every Request

Before any request reaches a Controller, the JwtFilter intercepts it. It reads the Authorization header, extracts the token, validates the signature and expiry. If valid, the request proceeds. If invalid or missing, it returns 403 Forbidden.

6

SecurityConfig Defines the Rules

/api/auth/** routes are public (permitAll) — anyone can login/register. All other routes require a valid JWT token (authenticated).

Security Files

FilePurpose
JwtUtil.javaToken factory — generateToken(), extractEmail(), isValid(). Creates and validates JWT tokens using HS256 signing.
JwtFilter.javaSecurity guard — intercepts every HTTP request, checks for valid Bearer token in Authorization header.
SecurityConfig.javaRules engine — defines which URLs are public (/api/auth/**) and which need authentication (everything else).
WebConfig.javaCORS config — allows React (port 5173) to make requests to Spring Boot (port 8080).
4 Controller Layer — REST APIs

Receives HTTP requests, processes business logic, returns JSON responses

What is a REST API?

REST (Representational State Transfer) is an architectural style for web APIs. Each URL represents a resource, and HTTP methods (GET, POST, PUT, DELETE) define what action to perform on that resource.

Spring Boot automatically converts Java objects to JSON in responses and JSON request bodies to Java objects.

Spring Boot Annotations Explained

@RestController Marks a class as a REST API controller. All methods return data (JSON), not HTML pages.
@RequestMapping("/api/x") Sets the base URL path for all endpoints in this controller.
@GetMapping Handles HTTP GET requests — used for fetching/reading data.
@PostMapping Handles HTTP POST requests — used for creating new data.
@PutMapping("/{id}") Handles HTTP PUT requests — used for updating existing data.
@RequestBody Tells Spring to read the JSON body of the request and convert it into a Java object automatically.
@RequestParam Reads query parameters from the URL. Example: /api/doctors?specialtyId=1&mode=ONLINE
@PathVariable Reads a variable from the URL path. Example: /api/appointments/{id}/cancel — id is the path variable.

5 Repository Layer — Spring Data JPA

Handles all database operations — you write zero SQL

How JpaRepository Works

Spring Data JPA is magic. You create an interface (not a class), extend JpaRepository, and Spring automatically provides all CRUD operations.

public interface UserRepository extends JpaRepository<User, Long> { User findByEmail(String email); } // That's it! You get ALL of these for free: // findAll() → SELECT * FROM users // findById(id) → SELECT * FROM users WHERE id = ? // save(user) → INSERT INTO users (...) VALUES (...) // deleteById(id) → DELETE FROM users WHERE id = ? // findByEmail(e) → SELECT * FROM users WHERE email = ? (custom)

Key Insight: Spring reads the method name and generates the SQL query automatically. findBySpecialtyIdAndMode() becomes WHERE specialty_id = ? AND mode = ?. You never write SQL.

6 Entity / Model Layer

Java classes that map directly to database tables using JPA annotations

JPA Annotations Explained

@Entity Tells JPA "this class represents a database table". Hibernate creates the table automatically.
@Table(name = "users") Custom table name. We use this for User because "user" is a reserved keyword in H2.
@Id Marks this field as the primary key of the table.
@GeneratedValue(IDENTITY) Auto-increment — the database generates the ID automatically (1, 2, 3...).
@ManyToOne Foreign key relationship. Many doctors belong to one specialty. Many appointments belong to one patient.
@JoinColumn(name = "x_id") Specifies the foreign key column name in the table.

7 H2 Database

In-memory relational database — zero setup required

What is H2?

H2 is a lightweight, in-memory SQL database written in Java. "In-memory" means data is stored in RAM, not on disk. When the server stops, all data is lost. When it restarts, tables are recreated from the @Entity classes.

For production, you'd switch to MySQL or PostgreSQL — just change one line in application.properties.

Configuration (application.properties)

# Database URL — "mem" means in-memory, "hackathon" is the DB name spring.datasource.url=jdbc:h2:mem:hackathon # Database driver spring.datasource.driver-class-name=org.h2.Driver # Auto-create/update tables from @Entity classes spring.jpa.hibernate.ddl-auto=update # Show SQL queries in console (useful for debugging) spring.jpa.show-sql=true # Enable H2 web console at /h2-console spring.h2.console.enabled=true # Server port server.port=8080

DataSeeder — Auto-creates Admin Account

On server startup, DataSeeder.java implements CommandLineRunner — a Spring interface that runs code automatically when the application starts. It checks if admin@demo.com exists, and if not, creates the admin user.

9 Complete User Flows

Step-by-step journey for both Patient and Admin

Patient Flow

1

Register

Patient fills in name, email, password → POST /api/auth/register → account created with role PATIENT.

2

Login

Enter email & password → POST /api/auth/login → server validates → returns JWT token → stored in localStorage.

3

Select Mode

Choose ONLINE (teleconsultation) or OFFLINE (in-clinic visit).

4

Browse Specialties & Doctors

GET /api/specialties → select one → GET /api/doctors?specialtyId=X&mode=Y → see matching doctors.

5

Pick Slot & Book

Select doctor → pick date → GET /api/slots → see available times → click Book → POST /api/appointments → slot locked, appointment CONFIRMED.

6

View & Manage Appointments

GET /api/appointments/my → see all bookings with doctor name, time, mode, status. Can cancel confirmed appointments (slot gets freed).

Admin Flow

1

Login as Admin

admin@demo.com / admin123 → gets JWT token with ADMIN role → redirected to /admin panel.

2

Add Specialties

POST /api/specialties → add Cardiology, Dermatology, Pediatrics, etc.

3

Register Doctors

POST /api/doctors → add doctors with name, specialty, mode (ONLINE/OFFLINE), video link or clinic address.

4

Create Time Slots

POST /api/slots → create available appointment slots for each doctor with date, start time, end time.

5

Manage Appointments & View Summary

Select date → see all appointments → mark as COMPLETED or NO_SHOW → view summary stats grouped by mode, specialty, and status.

What Happens During Booking (Detailed)

1. Patient clicks "Book" on a time slot 2. React sends: POST /api/appointments { slotId: 5, patientEmail: "john@email.com" } with header: Authorization: Bearer eyJhbGci... 3. Request hits Spring Boot server (port 8080) 4. CORS Filter → origin is localhost:5173 → ALLOWED 5. JWT Filter → reads token → validates signature → valid → PASS 6. AppointmentController.book() is called → slotRepository.findById(5) 7. Hibernate generates: SELECT * FROM SLOT WHERE ID = 5 → H2 returns slot data → converted to Java Slot object 8. Controller checks: slot.isBooked == false? → YES → proceed → sets slot.isBooked = true → saves slot (prevents double booking) → creates new Appointment (status = CONFIRMED, date = today) → saves appointment 9. Spring converts Appointment object → JSON response 10. React receives response → shows success alert → refreshes list

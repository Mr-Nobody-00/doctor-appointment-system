Complete Endpoint & Flow Guide

Every endpoint explained — what calls what, in what order, and what happens at each step.

How Every Request Works (The Chain)

Every single HTTP request from React to Spring Boot goes through the same chain of files in this exact order:

React (Browser) User clicks a button or page loads ↓ Axios sends HTTP request to http://localhost:8080/api/... Token is attached in header: Authorization: Bearer eyJhbGci... ↓ WebConfig.java — CORS Filter (Step 1) Checks: "Is this request from localhost:5173?" YES → Allow it through NO → Block it (CORS error) ↓ JwtFilter.java — Security Guard (Step 2) Checks: "Is this URL public? (/api/auth/**)" YES → Skip token check, go straight to controller NO → Read Authorization header → Extract token → Call JwtUtil.isValid(token) → VALID? → Set user in security context → continue → INVALID? → Return 403 Forbidden (BLOCKED) ↓ Controller — Business Logic (Step 3) The right controller method is called based on URL + HTTP method Example: POST /api/appointments → AppointmentController.book() ↓ Repository — Database Access (Step 4) Controller calls repository methods like findById(), save() Spring Data JPA generates SQL from the method name ↓ Hibernate ORM — SQL Generator (Step 5) Converts repository calls to actual SQL findByEmail("john@email.com") → SELECT * FROM users WHERE email = 'john@email.com' save(user) → INSERT INTO users (name, email, ...) VALUES ('John', 'john@email.com', ...) ↓ H2 Database — Data Storage (Step 6) Executes the SQL query Returns the result data ↓ Back to Controller (Step 7) Controller receives Java objects from repository Spring automatically converts Java objects to JSON ↓ Back to React (Step 8) Axios receives JSON response in .then(res => ...) React updates the UI with the new data

Remember this order for EVERY request:

React → CORS (WebConfig) → JWT Filter → Controller → Repository → Hibernate → H2 Database → back up the chain → JSON response → React

What Happens When the Server Starts
1

You run: mvnw spring-boot:run

Maven reads pom.xml, downloads all dependencies (Spring, H2, JWT library, etc), and compiles all Java files.

2

DemoApplication.java runs

@SpringBootApplication tells Spring to scan ALL files in com.example.demo and its subpackages. It finds all @Entity, @Repository, @Controller, @Configuration, @Component classes.

3

Spring reads application.properties

Connects to H2 in-memory database. Sets ddl-auto=update which means Hibernate will auto-create tables.

4

Hibernate scans @Entity classes

Reads User.java, Specialty.java, Doctor.java, Slot.java, Appointment.java. Creates the database tables: USERS, SPECIALTY, DOCTOR, SLOT, APPOINTMENT with all columns and foreign keys.

5

Spring registers security

SecurityConfig.java sets up the rules. WebConfig.java sets up CORS. JwtFilter.java is added to the filter chain.

6

DataSeeder.java runs (CommandLineRunner)

Checks if admin@demo.com exists. If not, creates the Admin user in the database. Prints "Admin user created!"

7

Tomcat starts on port 8080

Server is ready! "Started DemoApplication in X seconds" appears. Now it listens for HTTP requests.

Authentication Endpoints
POST /api/auth/register
Create a new patient account
PUBLIC — No token needed
What React Sends:
// User fills the Register form and clicks "Register" // Register.jsx calls: axios.post('http://localhost:8080/api/auth/register', { "name": "John Doe", "email": "john@email.com", "password": "mypassword" })
Step-by-Step Flow (which file does what):
1

Register.jsx (Frontend)

User types name, email, password. Clicks Register button. axios.post() sends JSON to backend.

2

WebConfig.java (CORS Check)

Request comes from localhost:5173. CORS config says "allow it". → PASS

3

JwtFilter.java (Security Check)

URL is /api/auth/register. SecurityConfig says /api/auth/** is permitAll(). → SKIP token check, allow through

4

AuthController.java → register() method

Spring sees POST + /api/auth/register → calls register(@RequestBody User user). Spring auto-converts the JSON body into a User Java object. Then sets user.setRole("PATIENT") — you can ONLY register as a patient, never as admin.

5

UserRepository.save(user)

Controller calls userRepository.save(user). Spring Data JPA passes this to Hibernate.

6

Hibernate → H2 Database

Hibernate generates: INSERT INTO users (name, email, password, role) VALUES ('John Doe', 'john@email.com', 'mypassword', 'PATIENT'). H2 executes it. User is saved with auto-generated id.

7

Response back to React

Controller returns the saved User object. Spring converts it to JSON. Register.jsx receives it in .then(), shows "Registered!" message, and redirects to login page after 2 seconds.

What React Receives:
{ "id": 2, "name": "John Doe", "email": "john@email.com", "password": "mypassword", "role": "PATIENT" }
POST /api/auth/login
Authenticate user and get JWT token
PUBLIC — No token needed
What React Sends:
// User fills email and password on Login page, clicks "Sign In" // Login.jsx calls: axios.post('http://localhost:8080/api/auth/login', { "email": "john@email.com", "password": "mypassword" })
Step-by-Step Flow:
1

Login.jsx (Frontend)

User enters email + password. Clicks Sign In. Axios sends POST request.

2

CORS + JWT Filter

CORS allows it. JWT Filter sees /api/auth/** → public route → skip token check. → PASS

3

AuthController.java → login() method

Reads email and password from request body. Calls userRepository.findByEmail(email) to find the user in the database.

4

Password Check

Compares the password from the request with the password stored in the database. If they don't match → throws "Invalid credentials" error.

5

JwtUtil.java → generateToken()

If password matches, calls JwtUtil.generateToken(email, role). This creates a signed JWT token containing the user's email and role, with 24-hour expiry.

6

Response back to React

Returns JSON with token, role, and name. Login.jsx stores them in localStorage and redirects — ADMIN goes to /admin, PATIENT goes to /dashboard.

What React Receives:
{ "token": "eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJqb2huQGVtYWlsLmNvbSIs...", "role": "PATIENT", "name": "John Doe" }
What Login.jsx does with the response:
// Store everything in browser's localStorage localStorage.setItem('token', res.data.token) // JWT token localStorage.setItem('role', res.data.role) // "PATIENT" or "ADMIN" localStorage.setItem('name', res.data.name) // "John Doe" localStorage.setItem('email', email) // "john@email.com" // Redirect based on role if (role === 'ADMIN') navigate('/admin') else navigate('/dashboard')

Why is this important?

After this point, EVERY future API call from React will include the token:
headers: { Authorization: "Bearer eyJhbGci..." }
Without this token, the backend returns 403 Forbidden.

Specialty Endpoints
GET /api/specialties
Get the list of all medical specialties
AUTHENTICATED — Token required
Step-by-Step Flow:
1

Dashboard.jsx or AdminDashboard.jsx

Page loads → useEffect runs → calls fetchSpecialties() → axios.get('/api/specialties', { headers }) with JWT token in header.

2

JwtFilter.java

Reads "Bearer eyJhbGci..." from header → calls JwtUtil.isValid(token) → token is valid → PASS

3

SpecialtyController.java → getAll()

Calls specialtyRepository.findAll() → Hibernate generates SELECT * FROM specialty → returns list of all specialties.

What React Receives:
[ { "id": 1, "name": "Cardiology" }, { "id": 2, "name": "Dermatology" }, { "id": 3, "name": "Pediatrics" } ]
POST /api/specialties
Add a new specialty (Admin only)
ADMIN — Token required
What React Sends:
axios.post('/api/specialties', { "name": "Cardiology" }, { headers })
Flow:

AdminDashboard.jsx → JwtFilter (validates token) → SpecialtyController.create() → specialtyRepository.save(specialty) → Hibernate: INSERT INTO specialty (name) VALUES ('Cardiology') → Returns the saved specialty with auto-generated id.

Doctor Endpoints
GET /api/doctors?specialtyId=1&mode=ONLINE
Get doctors filtered by specialty and mode
PATIENT — Token required
Step-by-Step Flow:
1

Dashboard.jsx

Patient selects "Cardiology" from dropdown + "ONLINE" mode → calls fetchDoctors(1, 'ONLINE') → Axios sends GET with query parameters.

2

JwtFilter.java → validates token → PASS

3

DoctorController.java → getBySpecialtyAndMode()

@RequestParam Long specialtyId reads "1" from the URL. @RequestParam String mode reads "ONLINE".

4

DoctorRepository.findBySpecialtyIdAndMode(1, "ONLINE")

Spring auto-generates: SELECT * FROM doctor WHERE specialty_id = 1 AND mode = 'ONLINE'. Returns only online cardiology doctors.

What React Receives:
[ { "id": 1, "name": "Dr. Sharma", "mode": "ONLINE", "videoLink": "https://meet.google.com/abc", "clinicAddress": null, "specialty": { "id": 1, "name": "Cardiology" } } ]
GET /api/doctors/all
Get ALL doctors (Admin panel needs this)
ADMIN — Token required

AdminDashboard.jsx loads this on page load. Calls doctorRepository.findAll() → returns every doctor regardless of mode or specialty.

POST /api/doctors
Register a new doctor (Admin only)
ADMIN — Token required
What React Sends:
axios.post('/api/doctors', { "name": "Dr. Sharma", "specialtyId": "1", "mode": "ONLINE", "videoLink": "https://meet.google.com/abc", "clinicAddress": "" }, { headers })
Flow:

DoctorController.create() → reads all fields from body → calls specialtyRepository.findById(1) to get the Specialty object → sets doctor.setSpecialty(specialty) → calls doctorRepository.save(doctor) → Hibernate: INSERT INTO doctor with specialty_id = 1.

Slot Endpoints
GET /api/slots?doctorId=1&date=2026-03-30
Get available (unbooked) time slots for a doctor on a date
PATIENT — Token required
Step-by-Step Flow:
1

Dashboard.jsx

Patient clicked Dr. Sharma, then picked date 2026-03-30 from the date picker → calls fetchSlots(1, '2026-03-30').

2

SlotController.java → getAvailableSlots()

Calls slotRepository.findByDoctorIdAndDateAndIsBooked(1, "2026-03-30", false)

The false is the key — it means "only give me slots where isBooked = false" = available slots only!

3

Hibernate generates SQL

SELECT * FROM slot WHERE doctor_id = 1 AND date = '2026-03-30' AND is_booked = false

What React Receives:
[ { "id": 1, "date": "2026-03-30", "startTime": "10:00", "endTime": "10:30", "isBooked": false }, { "id": 2, "date": "2026-03-30", "startTime": "11:00", "endTime": "11:30", "isBooked": false } ] // Booked slots are NOT returned — only available ones
POST /api/slots
Create a new time slot for a doctor (Admin only)
ADMIN — Token required
What React Sends:
axios.post('/api/slots', { "doctorId": "1", "date": "2026-03-30", "startTime": "10:00", "endTime": "10:30" }, { headers })

SlotController.create() → sets isBooked = false (new slot is always available) → links to doctor → saves to database.

Appointment Endpoints (Most Important!)
POST /api/appointments
Book an appointment — the core feature with double-booking prevention
PATIENT — Token required
What React Sends:
// Patient clicks "Book" on the 10:00-10:30 slot axios.post('/api/appointments', { "slotId": "1", "patientEmail": "john@email.com" }, { headers })
Step-by-Step Flow (this is the most important flow!):
1

Dashboard.jsx → bookAppointment(slotId)

Patient clicks "Book" button on a slot card. React sends POST with slotId and the patient's email (from localStorage).

2

JwtFilter.java → validates token → PASS

3

AppointmentController.java → book() method

Reads slotId from request body. Calls slotRepository.findById(1) to get the slot from database.

4

DOUBLE BOOKING CHECK (Critical!)

Checks if (slot.getIsBooked()) → throw "Slot already booked". If another patient already booked this slot, the request is REJECTED. This prevents two patients from booking the same time.

5

LOCK THE SLOT

Sets slot.setIsBooked(true) and saves it. Now this slot is marked as taken — no one else can book it. This is the "slot locking" the use case requires.

6

Find the patient

Calls userRepository.findByEmail("john@email.com") to get the User object.

7

Create the Appointment

Creates new Appointment object. Sets patient, doctor (from slot.getDoctor()), slot, status = "CONFIRMED", bookingDate = today. Saves to database.

8

Response back to React

Returns the full Appointment with all details. Dashboard shows "Appointment booked!" alert and refreshes the slot list (the booked slot disappears).

What React Receives:
{ "id": 1, "status": "CONFIRMED", "bookingDate": "2026-03-30", "patient": { "id": 2, "name": "John Doe" }, "doctor": { "id": 1, "name": "Dr. Sharma", "mode": "ONLINE", "videoLink": "https://meet.google.com/abc" }, "slot": { "id": 1, "date": "2026-03-30", "startTime": "10:00", "endTime": "10:30", "isBooked": true } }
GET /api/appointments/my?email=john@email.com
Get all appointments for the logged-in patient
PATIENT — Token required

Dashboard.jsx "My Appointments" tab → calls this → userRepository.findByEmail(email) → appointmentRepository.findByPatientId(patient.getId()) → returns all appointments for that patient with doctor, slot, and status info.

GET /api/appointments/all?date=2026-03-30
Get all appointments for a specific date (Admin view)
ADMIN — Token required

AdminDashboard.jsx "Appointments" tab → admin picks a date → calls appointmentRepository.findByBookingDate(date) → returns ALL appointments for that day from ALL patients.

GET /api/appointments/summary?date=2026-03-30
Get daily summary statistics grouped by mode, specialty, and status
ADMIN — Token required
Flow:

Gets all appointments for the date, then uses Java Streams with Collectors.groupingBy() to group them — like SQL's GROUP BY but in Java.

What React Receives:
{ "total": 8, "byMode": { "ONLINE": 5, "OFFLINE": 3 }, "bySpecialty": { "Cardiology": 4, "Dermatology": 2, "Pediatrics": 2 }, "byStatus": { "CONFIRMED": 5, "COMPLETED": 2, "CANCELLED": 1 } }

AdminDashboard.jsx displays this as stat cards — total appointments, count by online/offline, count by specialty, count by status.

PUT /api/appointments/5/cancel
Cancel an appointment — frees the slot for others
PATIENT — Token required
Step-by-Step Flow:
1

Dashboard.jsx

Patient clicks "Cancel" button on their appointment → calls axios.put('/api/appointments/5/cancel')

2

AppointmentController.cancel(5)

@PathVariable Long id reads "5" from the URL. Finds the appointment with id 5.

3

Set status to CANCELLED

appointment.setStatus("CANCELLED")

4

FREE THE SLOT (Very important!)

slot.setIsBooked(false) → saves the slot. Now this time slot is available again for other patients to book!

5

Save and respond

Saves the updated appointment. React refreshes the list — status changes from "CONFIRMED" to "CANCELLED".

PUT /api/appointments/5/status
Mark appointment as COMPLETED or NO_SHOW (Admin only)
ADMIN — Token required
What React Sends:
// Admin clicks "Complete" button axios.put('/api/appointments/5/status', { "status": "COMPLETED" }, { headers }) // OR Admin clicks "No Show" button axios.put('/api/appointments/5/status', { "status": "NO_SHOW" }, { headers })

AppointmentController.updateStatus() → finds appointment by id → sets the new status → saves. This completes the appointment lifecycle: CONFIRMED → COMPLETED or NO_SHOW.
